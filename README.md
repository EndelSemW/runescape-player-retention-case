##🧙 RuneScape Player Retention Case

## 📌 Overview (English)

This repository contains a **fictional but realistic analytics practice case** focused on **player retention analysis using panel-based, transaction-level data**, inspired by real-world take-home exercises commonly used in analytics interviews.

The dataset simulates **RuneScape player purchases across multiple cities** over two years (2021–2022) and includes **intentional data quality issues**, reflecting the challenges analysts often face when working with observational panel data.

This project is intended for **learning and training purposes** at an **Associate / Mid-Level Analytics** level.


# 🎯 Objectives

- Practice **retention analysis** and EDA on panel-based data.    
- Identify **data quality issues and limitations**. How might these factors affect how results should be interpreted?
- Identify any data irregularities or anomalies in the dataset.
- Explain what other information would be useful to extend this analysis. 
- Communicate findings clearly using any analytical tools of your choice, such as **SQL, Excel, Power Point, Power BI and Python**

## 📊 Dataset Notes

- The dataset is **transaction-level**. 
- A single `player_id` may appear multiple times within the same city and year.  
- Players may purchase different categories, in multiple cities, across years. 
- The dataset includes **intentional anomalies**, such as:
  - Extreme spend values  
  - Non-reliable panel start dates  
  - Temporal inconsistencies between panel start and purchase year  

Candidates are expected to **recognize and reason about these limitations**.

## 📌 Visão Geral (Português)

Este repositório contém um **case prático de analytics**, fictício mas realista, focado em **análise de retenção de jogadores usando dados transacionais de painel**, inspirado em testes técnicos reais do mercado.

O dataset simula **compras de jogadores de RuneScape em diferentes cidades**, ao longo de dois anos (2021–2022), e inclui **problemas intencionais de qualidade de dados**, refletindo desafios comuns em análises do mundo real.

Este material é voltado para **treinamento e estudo**, em nível **Associate / Pleno**.

## 🎯 Objetivos do Exercício

- Comunicar suas descobertas com clareza usando qualquer ferramenta de análise da sua escolha, como **SQL, Excel, Power Point, Power BI and Python**
